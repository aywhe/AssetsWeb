1. 问题：在虚拟机中，用nodejs配置后，外面的主机能访问，但是同样局域网的手机就不能访问。
方法：
    1） 使用桥接模式（vmnet0），将虚拟机直接暴露到局域网中，与其他设备一起使用一个网段 192.168.1.*。但是我试过，不行。
    2） 使用NAT模式（vmnet8），这也是虚拟机平时用的联网方式，只需要在配置一下就可以了。
        1） 编辑虚拟机，选择vmnet8，选择nat设置，端口转发，添加。
            这里可以将主机的端口映射到虚拟机的端口，这样手机访问主机的端口时，就会映射到虚拟机中。
            这里我的配置是 9998，tcp, 192.168.162.128:9998, 主机开发端口是9998，虚拟机网站端口也是9998，虚拟机在主机内的nat网络地址是 192.168.162.128。
        2） 还需要在主机防火墙中为 9998 端口开放。选择高级安全 Windows Defender 防火墙，选择入站规则，新建规则，端口，特定本地端口，tcp，.....
            这样，用手机访问主机 192.168.1.101:9998 就会被映射到虚拟机中
    完成。
2. 问题： 用 swiper 控件实现在网页中大量的图片的轮播显示，轮播一段世间后，网页崩溃，
        因为 html 文件中把服务端下发的大量图片路径都一时加入到 slide 的 src 里面，
        导致 swiper 里面有非常多的 slide 元素。
   方法：
       1） 初始的时候，只给 swiper 添加少量 slide。
       2） 做一个定时函数，隔一段世间，删除 swiper 的第一个 slide，添加另个图片 slide 到尾部，控制管理 swiper 的 slide 数量。
           使用到的代码:
           var wrapper = new Swiper('.swiper-container');
           wrapper.removeSlide(0);//(wrapper.firstElementChild);
           wrapper.appendSlide(slide);
3. 问题： 在调试 index.html 时，发现每次 swiper 的更改，都会自动生成一个 <span>，导致 swiper 自动播放一段时间后，网页上非常多的这个 <span> 元素。
        <span class="swiper-notification" aria-live="assertive" aria-atomic="true"></span>
        查找到 swiper-bundle.min.js 里面，原来是这一段代码在作怪：
        i("beforeInit",(()=>{l=v("span",t.params.a11y.notificationClass),l.setAttribute("aria-live","assertive"),l.setAttribute("aria-atomic","true")}))
    方法：
        问了 AI 助手这段代码的功能，主要是在初始化 Swiper 对象时会触发。
        原来是我在上次修改 swiper 的 slide 的数量时，每次都重新 new Swiper()，都会触发生成 <span>，
        所以我将 Swiper 对象做成全局变量了，就不用每次都 new 一个新的的了。
        代码：
        if(g_wrapper === null){g_wrapper = new Swiper('.swiper-container');}
4. 问题： 在<video>中不能播放mkv格式视频。
   方法： 将 <video><source></video> 中的 <source> 中的type设置为空。因为mkv只是容器，具体编码未知，所以让解码器自己判断。
5. 问题： 有的 mkv 文件播放的时候没有声音。
   方法： 似乎是因为浏览器不支持这些文件的音频解码。
6. 问题： 使用 <video><track></video> 添加 srt 格式字幕，但是不起作用。
   方法： 似乎是因为浏览器不支持 srt，而是支持 vtt 格式。
7. 问题： play.html 中 fetch('/api/lookfor-subtitle?path=' + (fullPath))，
        服务端app.get('/api/lookfor-subtitle', (req, res) => {const vitualPath = req.query.path;})
        发现 fullPath 中的 + 字符在vitualPath变成了空格字符
   方法： 改为  fetch('/api/lookfor-subtitle?path=' + encodeURIComponent(fullPath))，
        因为 + 字符本来就是空格字符的 URI encode，所以解码的时候解码成了空格字符。
8. 补充： 关于视频内嵌字幕和多音轨的问题，放弃自己在 JavaScript 上自定义提取，还是去找功能更强大的网页播放器更靠谱。
        外挂字幕现在只是默认支持 VTT 格式的，要支持其他格式，也是去找其他播放器更靠谱。
9. 问题： onclick="onClickFunction()"，出现错误 onClickFunction undefined。
   方法： function onClickFunction(){} 要放在 fetch 等异步函数操作的前面，反正就是要尽快加载 onClickFunction()
10. 问题： 自动播放功能在全屏时，跳转后不再全屏，所以传递参数自动全屏，但是遇到问题是，为了安全性考虑“全屏必需要用户点击”，不能用程序自动全屏。
   方法： 不跳转页面，只在当前页面修改播放参数。
11. 问题： play页面制作按键响应时，当video处于焦点时，video也会响应按键，方向键增加了两次进度条。
           还有，自定义空格键暂停，当video处于焦点时，也响应了两次。
    方法： 放弃页面按键响应，让用户聚焦video后让video自己响应按键。
12. 问题： play页面为了实现自动播放，使用“修改播放参数的方法”代替了“跳转页面的方法”以后，点击刷新，就会回到很久以前的视频，
          原因是页面还是从地址栏中获取视频文件地址。
    方法： 将更改后的视频文件地址存储到sessionStorage中，页面加载的时候读取sessionStorage是否有视频文件地址，再去读取地址栏。
          sessionStorage的生存期只有当前页面，所以可以这样用。
